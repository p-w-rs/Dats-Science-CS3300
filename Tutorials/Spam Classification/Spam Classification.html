Model for Spam Classification¶

In this tutorial, we look at classifying emails as either spam or ham. We use the trec07p data set. It contains about 61,000 emails recorded over a few months.

We will look at:

  • Representing text as numerical values using a bag of words model
  • Filtering out infrequent words and non-sense words and their effect on the number of features
  • Exploring high-dimensional data with dimensionality reduction
  • Clustering and finding features associated with clusters
  • Experimental setup for a supervised learning problem (classification)
  • Training a Logistic Regression model for classification
  • Evaluating the predictions from a classification model
  • Additional feature engineering to improve model accuracy

Import Data¶

In a previous notebook, I converted the messages from raw email files into JSON documents. Emails contain multiple parts including a header (key-value pairs such as the from, to, and subject fields) and a body / attachments. The body may come in text and/or HTML form. I extracted the bodies, and parsed any HTML if needed, so that the JSON documents contain plain text. I created one JSON file per message; the names of the files contain a unique message id. I also added the ham or spam label to each message.

An example message might look like this:

{ "from" : "\"RJ Nowling\" <rnowling@something.com>",
  "to" : "a_student@school.edu",
  "subject" : "Spam Classification",
  "body" : "Did you know you can buy a new iPhone?",
  "user-agent" : "Icedove 1.5.0.10 (X11/20070329)",
  "label" : 0,
  "date" : "Tue, 10 Apr 2007 19:25:38 +0000" }

A label of 0 indicates ham, while a label of 1 indicates spam.

# Find all json files in a directory, reads them one at a time,
# and returns the message id and deserialized JSON object

def stream_messages(dirname):
    for flname in glob.glob(dirname + "/*.json"):
        start = flname.rfind("_") + 1
        end = flname.rfind(".")
        message_idx = int(flname[start:end])
        with open(flname) as fl:
            message = json.load(fl)
            yield message_idx, message

def parse_date(date_string):
    """
    Parse a datetimes like
    
    'Tue, 10 Apr 2007 19:25:38 +0000'
    'Fri, 18 May 2007 12:15:02 -0700 (PDT)'
    '14 May 2007 04:27:55 +0000'
    '2007-06-02 17:35:21'nih
    
    """

    if date_string is None:
        return None
    
    # remove trailing "(GMT)"
    if date_string.endswith(")"):
        idx = date_string.rfind(" ")
        date_string = date_string[:idx]
    
    # remove preceding day of the week
    if "," in date_string:
        idx = date_string.find(",") + 2
        date_string = date_string[idx:]
    
    # remove timezone offset
    idx = date_string.rfind(" ")
    date_string = date_string[:idx]
        
    try:
        fmt = "%d %b %Y %H:%M:%S"
        return datetime.datetime.strptime(date_string, fmt)
    except:
        try:
            fmt = "%Y-%m-%d %H:%M:%S"
            return datetime.datetime.strptime(date_string, fmt)
        except:
            return None

Feature Engineering¶

We are going to engineer features from the message bodies. We will use a "bag of words" model. We identify all words across all documents and assign a unique index to each word. For each document, we create a vector of 0s and 1s, where a 1 indicates a word is present and 0 indicates a word is absent.

The vectorizer returns a sparse matrix. Many words are likely to only be present in a small handful of documents. This leaves a lot of zeros in the matrix. To avoid storing all of those zeros, a sparse matrix only stores the entries with non-zero values. Sparse matrices tend to be more memory efficient when fewer than 1/3 of entries are non-zero. Scipy provides several different forms of sparse matrix data structures. This is because not all sparse matrices are efficient for all types of operations (e.g., accessing data by row or column, matrix-vector multiplications, etc.).

We can often ignore words that are very rare (e.g., only occur in one or two documents). These words are unlikely to be useful since they don't generalize (occur across multiple documents). This reduces the number of features. We must be careful, however, that we don't exclude words that are somewhat rare as those can be informative. Let's look at the document frequencies of the words.

When manually looking at some of the messages, I noticed that some messages had "words" that were octal-encoded numbers (e.g., 0x5). These did not seem to be an encoding problem but certainly be difficult to interpret and lead to over-fitting in our model. Let's iterate through our vocabulary and set the counts for any octal words to 0.

Exploratory Data Analysis¶

We can't realistically assess around 35,000 features individually. We need to try alternative approaches. We'll try using dimensionality reduction to see if we the emails cluster. Ideally, we would hope the emails would cluster by ham or spam.

PCA is the most commonly used dimensionality reduction technique. Unfortunately, PCA as implemented in scikit-learn doesn't work on sparse matrices. We'll use SVD, which can be shown to be mathematically equivalent.

Inspecting Clusters¶

Now, we want to use statistical testing to find which words are associated with each cluster. We'll use a binomial test. Assume we flip two coins many times. The binomial test tells whether the two coins are weighted the same way by comparing the number of heads from one to the other. It gives us a p-value that we can use to perform a test of statistical significance.

For each word, we'll make a 2x2 table with absence / presence along one dimension and cluster along the other.

Cluster Absent Present
0 500 700
1 700 500

Since cluster 0 is larger, we'll use the counts for cluster 0 to calculate an expected success probability (present) with:

$$ p = \frac{cluster\_0\_present}{cluster\_0\_absent + cluster\_0\_present} $$

pvalues_greater = []

# occurs more frequently in cluster 0 than cluster 1
pvalues_less = []

n_words = body_features_train.shape[1]

# significance threshold
alpha = 0.001

# list of vocabulary words, ordered by column index
words = body_vectorizer.get_feature_names()

for word_idx in range(n_words):
    cluster_0_present = cluster_0_counts[0, word_idx]
    cluster_0_absent = cluster_0_docs.shape[0] - cluster_0_present

    cluster_1_present = cluster_1_counts[0, word_idx]
    cluster_1_absent = cluster_1_docs.shape[0] - cluster_1_present
    
    # expected probability based on cluster 0
    p = cluster_0_present / (cluster_0_absent + cluster_0_present)
    
    if p == 0.0:
        continue
    
    # run binomial tests
    pvalue_greater = binom_test([cluster_1_present, cluster_1_absent], p = p, alternative="greater")
    pvalue_less = binom_test([cluster_1_present, cluster_1_absent], p = p, alternative="less")

    # correct for multiple hypothesis testing (Bonferroni)
    corrected_pvalue_greater = pvalue_greater * n_words
    corrected_pvalue_less = pvalue_less * n_words
    
    if corrected_pvalue_greater < alpha:
        pvalues_greater.append((pvalue_greater, word_idx, words[word_idx], cluster_0_present, cluster_1_present))
        
    if corrected_pvalue_less < alpha:
        pvalues_less.append((pvalue_less, word_idx, words[word_idx], cluster_0_present, cluster_1_present))

Evaluating Additional Features: Sender Domains¶

Let's try using the domains. The from strings are like this:

Yan Morin <yan.morin@savoirfairelinux.com>

Let's extract the domain by using a regular expression that matches from the @ symbol until a character that is not a digit, letter, period, or dash.

In [52]:
regex = r"@([0-9a-zA-Z_.-]+)"
prog = re.compile(regex)

def extract_domain(from_address):
    match = prog.search(from_address)
    
    if not match:
        return ""
    else:
        # domain will be something like biostat.ku.dk
        domain = match.group(1).lower()
        
        # let's expand the domain by repeatedly chopping off the last parts: biostat.ku.dk biostat.ku biostat
        s = domain
        idx = domain.rfind(".")
        while idx != -1:
            domain = domain[:idx]
            s += " " + domain 
            idx = domain.rfind(".")
    
    return s
